7. JSP(Java Server Page)

✔️ Servlet의 변형 / HTML 코드에 자바 코드를 삽입하는 형태로 사용할 수 있는 Java Web component

✔️ 최근의 web Programming 에서 MVC 패턴이나 Model2 패턴이 등장

→ jsp 파일에 java 코드(스크립트릿 - java 로직, 표현식 - java 데이터 직접 출력)을 직접 사용하는 것을 금기시

MVC(Model view Controller) 패턴 : 역할 별로 분리 작성

Model : 데이터를 만들어내는 부분 - Service, Repository, VO-DTO-entity등

View : 화면 출력 부분 - JSP

controller 2개를 연결 시켜주는 부분 - Servlet

✔️ 한동안은 JSP 파일에 EL과 JSTL을 이용해 출력

JSP : Server 측에서 해석 → HTML 코드를 클라이언트에 전송

최근 : 서버에서 렌더링을 하지 말자는 추세 → JSP 사용률 저하

이유 : 클라이언트 디바이스의 다양화 ⇒ 모든 클라이언트의 요청에 맞추어서 화면 구성이 힘듦

✔️ 그래서 서버에서 REST API 형태로 json 형식의 데이터를 전송

→ 클라이언트 측: web socket, ajax, FETCH API등을 이용해서 데이터를 전송

→ 전송 받은 데이터를 파싱 해서 직접 출력

✔️ 이러한 작업을 단순화해주는 라이브러리 : React / Vue / Angular

1) 내장 객체 HttpServletRequest request:

✔️ 클라이언트의 요청 정보를 저장한 클래스의 객체

✔️ jsp에서는 내장이고 Servlet 에서는 요청 처리 메서드의 첫 번째 매개변수

✔️ 이 객체를 이용해서 클라이언트의 정보를 읽음

✔️ 클라이언트의 정보 중에서 가장 대표적인 것 : 요청 방식, 요청 URL, 파라미터 등

  • 요청 방식
    • GET :

      🔹데이터를 URL에 붙여서 전송하는 방식

      🔹URL 뒤에 ?를 추가하고 속성이름=값&의 형태로 전송(query String 이라고도 함)

      🔹파라미터의 길이에 제한 ⭕

      🔹보안성 ⬇️

      🔹자동 재 전송 기능 ⭕

      🔹일반적으로 읽기 작업에 사용

    • POST :

      🔹데이터를 요청 헤더에 숨겨서 전송하는 방식

      🔹길이에 제한 ❌ 외부로 노출 ❌ → 보안성 ⬆️

      🔹자동 재 전송 기능 ❌

      🔹전에는 삽입, 삭제, 갱신 등에 모두 이용했지만 최근에는 삽입에만 이용하는 것을 권장

      🔹반드시 인코딩을 해서 전송하고 읽는 곳에서 인코딩 설정을 해서, 읽어야 한다면 전송하고자 하는 데이터에 파일이 있거나 textarea, password가 있다면 GET으로 전송 ❌

      🔹파일을 전송할 때 enctype을 multipart/form-data로 설정

    • PUT : POST와 동일 (수정 시 사용)
    • DELETE : POST와 동일(삭제 시 사용)

  • 파라미터

    ✔️ 파라미터를 읽을 때는 request.getParameter / request.getParameterValues 메소드 이용

    ✔️ Spring에서는 대부분 파라미터를 VO나 DTO class를 이용해서 읽음

    ✔️ 최근에는 하나의 파라미터 같은 경우는 파라미터를 이용하지 않고 URL 이용

    ✔️ request 객체 이용 → 접속한 클라이언트의 IP나 referrer나 운영체제 종류 및 브라우저 종류를 파악 가능

    ✔️ HttpServletResponse response :

    요청 처리 메소드의 2번째 매개변수 ⇒ 응답 객체로 클라이언트에게 응답을 전송할 때 사용

    ✔️ JSPWriter out:

    🔹 Servlet에서는 response를 이용해 생성

    🔹 클라이언트의 브라우저에 출력하기 위해서 버퍼에 기록하는 클래스의 인스턴스

    ✔️ ServletContext application: 모든 클라이언트가 공유할 수 있는 객체

    ✔️ HttpSession session

    🔹 request 객체를 이용해서 생성 가능

    🔹 하나의 클라이언트가 사용할 수 잇는 객체로 접속할 때 만들어지고 접속이 끝나면 소멸

    ✔️ Object page : 하나의 페이지가 생성될 때 만들어지고 페이지가 없어질 때 소멸되는 객체

    ✔️ ServletConfig config : 서블릿 환경 설정 정보

    ✔️ PageContext pageContext: JSP 페이지 내에서 사용할 수 있는 데이터 저장 기능을 소유한 객체

    ✔️ Throwable exception: 예외 발생했을 때 예외 내용을 가지고 있는 객체

2) 서버 측 요청 흐름 제어

⇒ forward:

현재 요청을 종료하지 않고 이어나가는 흐름

request와 session그리고 application 객체 모두 유지가 됩니다.

url이 변경되지 X

읽기 작업에서 결과 페이지로 이동할 때 주로 사용

⇒ redirect :

현재 요청을 종료하고 새로운 흐름을 만드는 것

request 객체를 새로 만들어지고 session 그리고 application 객체 모두 유지됨

url이 변경

삽입,삭제,갱신 작업을 수행할 때 주로 하지만 트래픽 제어를 위해서 읽기 작업에서 사용하기도 함
