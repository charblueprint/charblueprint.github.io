2. Web Service 구현 방식

1) 정적 어플리케이션 (HTML, CSS, JavaScript)

자바 스크립트를 이용해서 동적 구현 가능

JavaScript: 자신의 브라우저 바깥으로 나가지 못함

2) 동적 어플리케이션

CGL(common Gateway Interface): 클라이언트의 요청을 별도의 프로그램이 처리하는데 이 경우에는 프로세스 형태로 처리 - 한번의 하나의 요청만 처리 가능한 방식으로 Perl이나 VB 방식의 asp

Application Server 방식 : 클라이언트의 요청의 별도의 프로그램이 처리하는데 스레드 형태로 처리

→ 한 번에 여러 개 처리가 가능한 방식으로 java, C#, Python, Ruby, JavaScript의 Framework 등을 이용해서 구현

✔️ Web Server 이 외에 프로그램을 호출할 별도의 컨테이너가 필요

✔️ 이 경우에는 일반적으로 Web Server, Web Container, Application Server Program, Repository로 구성됨

  • Web Server : Apache, IIS, Linux Web Server
  • Web Container : Tomcat, Web Logic, Jboss, Zetty, resion (국산 제품: zeus)
  • Application Server Program : 프로그래밍 언어만 / Framework을 같이 사용
  • Repository : RDBMS(Oracle, MYSQL, MS-SQL), NoSQL, Hadoop 등의 분산 처리 시스템)
